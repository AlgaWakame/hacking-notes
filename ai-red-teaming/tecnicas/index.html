    Técnicas de AI Red Teaming¶

    Guía rápida para consultar técnicas prácticas de reconocimiento, explotación y validación en sistemas integrados con IA.

    Esta sección está pensada para consulta durante laboratorios, preparación de examen y auditorías autorizadas.

    Flujo recomendado¶

    Reconocimiento → Superficie IA → RAG → Agentes/Herramientas → Prompt Injection → Evidencia

    Técnicas¶

    Técnica Uso principal
    Reconocimiento Mapa rápido de reconocimiento IA
    Reconocimiento pasivo Headers, health/status, docs API, repositorios y fuzzing controlado
    Reconocimiento activo Modelo, OpenAI-compatible API, RAG, MCP/tools, agentes y detección
    Prompt Injection Directa Probar si el usuario puede alterar el comportamiento del modelo
    Prompt Injection Indirecta Probar instrucciones maliciosas en documentos, páginas o fuentes externas
    RAG Data Extraction Validar fugas de documentos, citas, chunks o datos de otros usuarios
    SSRF vía Agente Revisar herramientas que procesan URLs controladas por usuario
    Tool Abuse Validar abuso de herramientas, funciones, conectores o acciones externas

    Checklist rápida¶

    [ ] Identificar endpoint principal
[ ] Revisar headers
[ ] Revisar health/status
[ ] Confirmar si hay API compatible con OpenAI
[ ] Buscar modelo/proveedor
[ ] Buscar RAG/vector DB
[ ] Buscar MCP/tools/agentes
[ ] Revisar repositorios/configs
[ ] Probar comportamiento activo con prompts controlados
[ ] Guardar request/response
[ ] Separar hechos observados de hipótesis
